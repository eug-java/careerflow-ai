# CareerFlow AI - Demo Script

## Goal

This demo shows an end-to-end AI-powered job application workflow:

```text
Login
Create profile
Create job manually or parse job description with AI
Generate resume or cover letter
Track workflow status
Preview generated document
Download PDF or DOCX
Delete data
```
Total demo time: 5-7 minutes.

# 1. Start the Application
   Start infrastructure
```bash
   docker compose up -d
```
   Start backend services

Recommended order:

- auth-service
- profile-service
- job-service
- matching-service
- document-service
- ai-generation-service
- workflow-service
- api-gateway-service

# 7. Generate Document

Go back to:

**/jobs**

Select profile.

Click one of:
```text
Generate Cover Letter
Generate Resume
```
Show:
```text
Workflow Status: RUNNING
Workflow Status: COMPLETED
```
Explain:

The frontend starts a Camunda workflow through api-gateway-service.
workflow-service starts a process instance.
A worker calls ai-generation-service.
Another worker publishes a Kafka event.
document-service consumes the event and stores the document in MinIO.
The frontend receives workflow status updates through WebSocket with polling fallback.

# 15. Common Demo Issues

### Login fails

Check:
```bash
curl -i -X POST http://localhost:8080/api/v1/auth/login \
-H "Content-Type: application/json" \
-d '{"username":"demo","password":"demo"}'
```
### CORS issue

Check gateway CORS and OPTIONS permit.

### AI parsing fails

Check:
```bash
echo $OPENAI_API_KEY
```
Start service with:
```bash
OPENAI_API_KEY=$OPENAI_API_KEY mvn spring-boot:run
```
### Kafka issue

Check Kafka topic:
```bash
docker exec -it careerflow-kafka /opt/kafka/bin/kafka-topics.sh \
--bootstrap-server localhost:9092 \
--list
```
### Document not visible

Check:
```bash
curl http://localhost:8080/api/v1/documents \
-H "Authorization: Bearer TOKEN"
```
